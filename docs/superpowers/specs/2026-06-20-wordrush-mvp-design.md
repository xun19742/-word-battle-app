# WordRush MVP 设计文档

## 1. 项目目标

WordRush 是一个原生微信小程序。首个 MVP 聚焦单人背词闭环，提供单词卡片和四选一两种学习模式，并使用微信云开发保存正式学习数据。

排行榜与联机抢答不进入本阶段，只保留未来扩展所需的用户标识、积分和学习记录边界。

## 2. 技术方案

- 客户端：原生微信小程序，使用 WXML、WXSS、JavaScript 和 JSON。
- 后端：微信云开发 CloudBase。
- 身份：云函数获取用户 OpenID，不自建账号密码体系。
- 数据库：云数据库保存用户、词书、单词和学习记录。
- 本地存储：仅保存尚未同步的本轮答题结果和必要设置缓存；云数据库是正式数据源。
- 代码要求：新增业务代码使用中文注释，注释解释业务意图，不重复描述语法。

## 3. MVP 功能范围

### 3.1 本期实现

1. 微信云开发登录。
2. 内置“四级核心词汇”词书，共 100 个单词。
3. 首页展示当前词书、今日进度、默认学习模式和开始学习入口。
4. 单词卡片模式：查看单词与音标、翻开释义、选择“认识”或“不认识”。
5. 四选一模式：从四个中文释义中选择答案，立即反馈正误并展示例句。
6. 每轮默认学习 10 个单词。
7. 学习总结展示得分、正确数、错误数和正确率。
8. 设置页切换默认学习模式。
9. 错词记录与复习入口。
10. 网络失败提示、待同步结果缓存和恢复后重试。

### 3.2 本期不实现

- 排行榜。
- 实时房间、匹配、联机抢答和断线重连。
- 好友、聊天和社交分享。
- 付费、广告和运营后台。
- 多端发布。
- 用户自定义上传词书。

## 4. 页面与交互

### 4.1 首页

首页读取用户设置和今日学习汇总，展示当前词书“四级核心词汇”、今日目标 10 词、已完成数量和默认模式。

“开始背词”直接进入默认模式。设置入口位于右上角。对战模式只显示“后续开放”，不提供可点击的伪功能。

### 4.2 单词卡片页

页面先展示英文单词、音标和发音按钮。用户翻开卡片后看到中文释义、英文例句和中文翻译，再选择“认识”或“不认识”。

“认识”记为正确，“不认识”记为错误并加入错词记录。完成 10 词后进入学习总结。

### 4.3 四选一页

页面展示英文单词和四个中文释义。选项由一个正确释义和三个同词书干扰项组成，顺序随机且不得重复。

选择后立即锁定本题，显示正确答案、例句和下一题按钮。答错的单词加入错词记录。完成 10 题后进入学习总结。

### 4.4 学习总结页

展示本轮得分、正确数、错误数、正确率，并提供“再练一轮”“复习错词”和“返回首页”。

### 4.5 设置页

设置项包括默认学习模式和每轮单词数。MVP 默认卡片模式、每轮 10 词；每轮单词数仅允许 10 或 20。

设置写入云端，并同步到本地作为启动缓存。

## 5. 数据设计

### 5.1 users 集合

```js
{
  _id: "云数据库文档 ID",
  _openid: "微信用户 OpenID",
  nickname: "微信昵称或默认昵称",
  avatarUrl: "头像地址或空字符串",
  defaultMode: "flashcard", // flashcard 或 quiz
  roundSize: 10, // 10 或 20
  totalScore: 0,
  createdAt: "数据库服务器时间",
  updatedAt: "数据库服务器时间"
}
```

### 5.2 wordbooks 集合

```js
{
  _id: "cet4-core-100",
  name: "四级核心词汇",
  description: "适合 MVP 体验的四级核心词汇",
  wordCount: 100,
  enabled: true
}
```

### 5.3 words 集合

```js
{
  _id: "单词文档 ID",
  wordbookId: "cet4-core-100",
  word: "adventure",
  phonetic: "/ədˈventʃər/",
  meaning: "n. 冒险；奇遇",
  example: "Life is a great adventure.",
  exampleTranslation: "生活是一场伟大的冒险。",
  difficulty: 1,
  order: 1
}
```

### 5.4 learning_records 集合

每个用户、词书和单词只保留一条累计记录，避免重复文档导致统计困难。

```js
{
  _id: "云数据库文档 ID",
  _openid: "微信用户 OpenID",
  wordbookId: "cet4-core-100",
  wordId: "单词文档 ID",
  correctCount: 0,
  wrongCount: 0,
  lastResult: "wrong", // correct 或 wrong
  isWrongWord: true,
  lastMode: "quiz", // flashcard 或 quiz
  lastStudiedAt: "数据库服务器时间",
  updatedAt: "数据库服务器时间"
}
```

## 6. 数据流

1. 小程序启动并初始化云开发。
2. 调用登录云函数获取 OpenID，并读取或创建 users 文档。
3. 首页并行读取用户设置、词书信息和当日学习汇总。
4. 开始学习时，从 words 中按词书读取候选单词，优先抽取未学习词和错词。
5. 客户端维护本轮答题状态；每题结束后更新内存状态。
6. 每轮结束时批量提交学习结果，更新 learning_records 和用户总分。
7. 提交失败时把本轮结果写入本地待同步队列，并明确提示用户。
8. 下次启动或网络恢复时重试队列；使用本轮唯一标识避免重复计分。

## 7. 错误处理

- 云环境未配置：展示可理解的配置提示，不进入空白页。
- 登录失败：允许用户点击重试，不伪造已登录状态。
- 词库为空：展示“词书数据未初始化”，并阻止开始学习。
- 单词不足四个：四选一模式提示数据异常，不生成重复选项。
- 提交失败：保留本轮结果到本地队列，成功同步后再移除。
- 重复点击答案：首次选择后锁定题目，后续点击不重复计分。
- 页面退出：保留当前轮次到本地，返回后可继续。

## 8. 测试与验收

### 8.1 逻辑测试

- 四选一选项包含且仅包含一个正确答案。
- 四个选项不重复。
- 卡片模式“认识/不认识”正确映射学习结果。
- 答题后不能重复计分。
- 设置仅接受合法模式和每轮数量。
- 待同步队列失败后保留、成功后清除。
- 同一轮结果重复提交不会重复累计积分。

### 8.2 手工验收

在微信开发者工具中验证：

1. 首次启动可以创建用户并进入首页。
2. 内置词书显示为 100 词。
3. 两种模式均可完成 10 词学习。
4. 设置切换后，下次开始学习进入新默认模式。
5. 答错单词出现在错词复习中。
6. 学习总结数据与本轮操作一致。
7. 关闭网络后提交会提示失败，恢复后可以成功同步。
8. 重新启动小程序后正式学习数据仍存在。

## 9. 实施顺序

按可运行功能逐步交付：

1. 项目骨架与云开发初始化。
2. 登录和用户数据。
3. 内置词书与词库初始化。
4. 首页与学习轮次服务。
5. 单词卡片模式。
6. 四选一模式。
7. 学习总结与云端同步。
8. 设置。
9. 错词复习。
10. 完整回归和说明文档。

每个功能完成后单独测试和提交，不在同一提交中混入后续功能。
