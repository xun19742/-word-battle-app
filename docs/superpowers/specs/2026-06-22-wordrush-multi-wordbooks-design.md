# WordRush 多词书与每日学习计划设计

## 目标

为 WordRush 增加四级、六级、考研、雅思四本内置公共词书，并允许用户切换词书、设置每日新词量和复习比例。功能必须离线可用，切换词书不能混淆或丢失学习记录。

本设计只包含公共内置词书。用户个人 CSV 词书导入是下一期独立功能，避免把公共数据生成、个人文件解析和学习计划迁移同时塞进一次实现。

## 数据来源与许可

内置词书从 [ECDICT](https://github.com/skywind3000/ECDICT) 生成。ECDICT 使用 MIT 许可证，提供音标、中文释义、考试标签和词频信息。

- 固定来源提交：`bc015ed2e24a7abef49fc6dbbb7fe32c1dadaf8b`
- 来源提交日期：2025-03-28
- 标签映射：`cet4`、`cet6`、`ky`、`ielts`
- 仓库中增加第三方声明，保留 ECDICT 项目名称、链接、固定提交和 MIT 许可证文本。
- 生成器只读取固定来源数据，不在运行时访问第三方网络。

ECDICT 的大量词条没有例句。`example` 和 `exampleTranslation` 改为可选字段，页面在字段为空时隐藏例句区域，不显示空白占位，也不生成不可靠的虚构例句。

## 紧凑词书结构

四本考试词书存在大量重复单词。生成器不为每本词书复制完整词条，而是输出：

- 一份共享词条表：单词、音标、中文释义、难度和稳定 ID。
- 四份词书索引：每本书只记录共享词条表中的下标。
- 一份词书清单：词书 ID、名称、描述、词数和来源标签。

共享词条按词频优先级排序；没有有效词频时按单词字母顺序排序。词条 ID 由规范化单词生成，同一个单词在不同词书中复用同一词条 ID。

生成脚本必须验证：

- 四个标签均能生成非空词书。
- 单词、音标和中文释义完整；无法提供音标的词条允许保留，但中文释义不能为空。
- 同一本词书内没有重复单词。
- 词书索引全部指向有效共享词条。
- 生成文件不超过 1.8 MB；超出时构建失败，避免微信小程序主包无意膨胀。

生成结果提交到仓库，因此用户导入项目后无需联网或重新生成即可使用四本词书。原有“四级核心词汇 100”保留为兼容入口，但不再作为唯一词书。

## 设置模型

设置从单一 `roundSize` 升级为：

```js
{
  defaultMode: 'flashcard',
  selectedWordbookId: 'cet4',
  dailyNewWords: 25,
  reviewRatio: 2
}
```

- `dailyNewWords` 范围为 5 到 500，默认 25，以 5 为步长。
- 设置页同时提供滑块与数字输入；非法值保存时会限制到范围内并对齐到最接近的 5。
- `reviewRatio` 只允许 1、2、3，分别表示新词与复习词为 `1:1`、`1:2`、`1:3`。
- 默认 `reviewRatio` 为 2，因此默认每日复习目标为 50。
- 每日复习目标通过 `dailyNewWords * reviewRatio` 实时计算，最大为 1500。
- 每轮学习仍最多 10 词；每日目标通过多轮学习完成。

旧设置只有 `defaultMode` 和 `roundSize` 时，保留 `defaultMode`，使用新的默认词书、每日新词量和复习比例。旧 `roundSize` 不再控制每日目标。

## 首页与词书选择

首页显示当前词书，并提供进入词书选择页的入口。词书选择页列出四本考试词书和兼容词书，展示词数、来源和当前选择状态。

首页将原有单一进度改为两组：

- 今日新词：已完成数量 / 每日新词目标。
- 今日复习：已完成数量 / 自动计算的复习目标。

首页提供两个明确入口：

- “学习新词”：只选择当前词书中尚无学习记录的单词。
- “今日复习”：只选择当前词书中已有学习记录的单词，错词优先，其次选择最久未学习的单词。

每次最多创建 10 词轮次。剩余目标少于 10 时按剩余数量创建；当今日目标完成、没有新词或没有可复习单词时，页面显示明确状态并阻止创建空轮次。

切换词书不会删除当前活动轮次。继续学习入口显示活动轮次所属词书，用户可以完成旧轮次后再开始新词书。

## 学习记录与数据迁移

学习记录键从单独 `wordId` 升级为 `wordbookId:wordId`，记录增加：

- `wordbookId`
- `studyType`：`new` 或 `review`
- `lastStudiedAt`
- 现有正确次数、错误次数、错词状态和最后模式

轮次与总结增加 `wordbookId` 和 `studyType`。本地每日汇总增加：

```js
{
  date: '2026-06-22',
  newCompleted: 0,
  reviewCompleted: 0,
  completed: 0,
  score: 0
}
```

`completed` 继续保留，兼容现有首页与云同步数据。旧记录缺少 `wordbookId` 时迁移到 `cet4-core-100`；旧日汇总缺少新词与复习字段时补零，不猜测历史轮次类型。

云端 `users` 设置同步新增选中词书、每日新词量和复习比例。`learning_records` 与 `learning_rounds` 保存词书和学习类型，移除硬编码的 `cet4-core-100`。云端继续接受旧轮次，缺失字段时按兼容词书和 `new` 类型处理。

## 服务边界

- 词书生成器：读取 ECDICT CSV，筛选标签、清洗数据、去重并生成紧凑数据文件。
- 词书服务：列出词书、读取指定词书、验证词书 ID，不关心页面状态。
- 学习计划服务：根据词书、学习记录、每日剩余目标和学习类型选择下一轮单词。
- 设置服务：标准化和保存词书 ID、每日新词量、复习比例与默认学习模式。
- 学习仓库：按词书隔离记录，聚合每日新词和复习完成量。
- 页面：只负责展示词书、目标和操作结果，不复制筛选算法。

## 异常处理

- 词书 ID 无效时回退到四级词书。
- 生成数据缺少中文释义或索引越界时构建失败，不提交损坏词书。
- 没有可用新词或复习词时显示提示，不覆盖已有活动轮次。
- 本地旧记录格式异常时回退到空状态，不影响应用启动。
- 云同步失败时继续使用本地学习计划，并沿用现有离线队列重试。
- 例句缺失时只隐藏例句区域，不影响答题和结果记录。

## 测试与验收

- 用小型 ECDICT 固定样例测试四类标签筛选、共享词条去重、排序和稳定 ID。
- 验证生成数据的四本词书非空、无重复、索引有效和包体大小门槛。
- 验证默认设置为新词 25、复习比例 2，并覆盖 5、500、非法步长和非法比例。
- 验证新词选择排除已学单词，复习选择优先错词和最久未学习单词。
- 验证每日目标拆分为最多 10 词的轮次，并正确处理目标完成和词量不足。
- 验证相同单词在不同词书的学习记录互不影响。
- 验证旧设置、旧记录和旧轮次可以迁移。
- 验证缺失例句的单词在卡片页和四选一页正常显示。
- 在微信开发者工具中手工切换四本词书，完成新词与复习轮次，确认首页进度和设置即时更新。

## 非目标

- 本期不提供个人 CSV、Excel 或 JSON 文件上传。
- 本期不引入完整间隔重复算法；复习顺序使用“错词优先、最久未学习优先”。
- 本期不为数万词自动生成例句、图片或音频。
- 本期不修改排行榜或联机对战功能。
